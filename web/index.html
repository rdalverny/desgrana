The situation

When a Behringer Wing or X32 records a multitrack session to disk (via a W-Live or X-Live card), it writes a single interleaved multichannel WAV: one file, all your channels.

Most DAWs (Logic Pro, Reaper, Pro Tools, Ableton Live) cannot open it directly as separate tracks: you get a stereo import, or nothing useful at all.

Desgrana reads that file, splits each channel into its own mono or stereo WAV, picks up the channel names from your console snapshot, and drops everything into a folder your DAW can load immediately.

What Desgrana does

  • Drag and drop your session folder, open your DAW, start mixing.
  • Your files come out exactly as recorded: no conversion, no resampling.
  • Channel names and stereo pairs from your console snapshot (KICK, SNARE, VOX…) are picked up automatically.
  • Only the channels you actually recorded show up. No empty tracks to clean up manually.
  • If you used markers in your recording session, they land in your DAW, as cue chunks in the WAV or MIDI file.
  • (macOS) signed and notarized.
  • Works entirely offline.
  • Open Source, MIT license.

FAQ

Why can't I open my Wing or X32 recording directly in Logic Pro or Reaper?

When recorded via a W-Live or X-Live card, the Wing and X32 write a single interleaved multichannel WAV — one file containing all channels. Logic Pro imports it as stereo; Reaper can split it with "Explode multichannel audio" but leaves you with numbered empty tracks to rename.

Desgrana automates the split, removes silent channels, and names each output file after your console channel.

How do I install Desgrana?

(macOS) Open the DMG archive, drag Desgrana into the Applications folder, and voilà!

(Linux) Download the .deb or .rpm for your system and install it.

Which platforms are supported?

Desgrana is available for macOS 13+ and Linux (Debian/Ubuntu 12+, Fedora 41+) as of version 1.8.

No Windows version is planned as of now. See CONTRIBUTING.

How do I get separate WAV tracks from a Behringer Wing or X32 recording?
Drop the session folder onto Desgrana. It reads the multichannel WAV (and the console snapshot if present — .snap for Wing, .scn for X32), then writes one WAV per channel, named and bounded, with markers embedded; ready to import into any DAW.

Do I need the .snap or the SE_LOG.bin file?

Neither is strictly required — Desgrana still extracts every channel into a numbered mono file without them.

SE_LOG.bin carries the channel count, sample rate, and marker positions. Without it, markers are not exported and Desgrana infers the channel count from the WAV file itself.

The snapshot (.snap for Wing, .scn for X32) is used to name output files after your console channels and to detect USB stereo pairs and channel names. If it is in the same session folder, it is picked up automatically. LCL stereo pairs are detected from channel names (L/R suffixes).

My session is split across several WAV files: does that work?

Yes, the console creates a new file each time the previous one hits 4 GB (FAT32 limit). Desgrana processes the take files in order and writes one continuous output per channel.

Are session markers preserved?

Markers are embedded as standard cue chunks in every output WAV file, and also exported as a CSV, a plain-text file, and a Standard MIDI File (type 0, SMPTE timecode).

Most DAWs can import the MIDI file to place markers at the correct positions, independent of the project tempo.

Does it send any data online?

No, it works entirely offline.

The only outbound request is an update check once a month, that you may disable in the app settings.

Can I rename tracks before extracting them?

Track names come from your console snapshot (.snap for Wing, .scn for X32/M32). The right place to fix them is either on the console itself, or in your DAW after import.

Desgrana's job is to translate the multitrack WAV into separate, ready-to-use tracks, not to edit the session.

I record directly into Reaper from my console — do I need Desgrana?

Not for that session: Reaper records individual tracks directly, already split.

Desgrana is useful if you also run a W-Live or X-Live as a simultaneous backup, if you receive sessions recorded by someone else on a Wing or X32 without a computer, or if you need to access archived sessions from a previous show.
