The problem situation

The Behringer Wing records everything into a single multichannel WAV file: one file, all your channels interleaved.

Most DAWs (Logic Pro, Reaper, Pro Tools, Ableton Live) cannot open it directly as separate tracks: you get a stereo import, or nothing useful at all.

Desgrana reads that file, splits each channel into its own mono or stereo WAV, picks up the channel names from your Wing snapshot, and drops everything into a folder your DAW can load immediately.

What Desgrana does

  • Drag and drop your session from your SD card, *click*, *click*, DAW.
  • Works entirely offline.
  • Your files come out exactly as recorded: no conversion, no resampling.
  • Channel names and stereo pairs from your Wing snapshot (KICK, SNARE, VOX…) are picked up automatically.
  • You can adjust stereo pairs before extraction.
  • Runs fine on a laptop in a backstage.
  • Only the channels you actually recorded show up. No empty tracks to clean up manually.
  • If you used markers in your recording session, they land in your DAW, as cue chunks in the WAV or MIDI file.
  • (macOS) signed and notarized.
  • Open Source (MIT license).

FAQ

Why can't I open my Wing recording directly in Logic Pro or Reaper?

The Wing writes a single interleaved multichannel WAV. One file containing all recorded channels. Logic Pro imports it as stereo; Reaper can split it manually but leaves you with empty tracks to clean up.

Desgrana automates the split, removes empty channels, and names each file after your console channel.

How do I install Desgrana?
(macOS only) Open the DMG archive, drag and drop Desgrana into the Applications folder, and voilà!
Is there a Windows / Linux version?

Currently, Desgrana is only available for macOS 13 and later on Apple Silicon or Intel.

A native Linux version (packaged for Debian/Ubuntu) is almost ready to test.

A native Windows version may follow soon after during Summer 2026. It is not sure yet which versions will be supported.

If you want to participate in testing it, please reach out.

How do I get separate WAV tracks from a Behringer Wing SD card recording?
Drop the session folder onto Desgrana. It reads the multichannel WAV (and the .snap snapshot if present), then writes one WAV per channel, named, trimmed, with markers embedded; ready to import into any DAW.

Do I need the .snap or the SE_LOG.bin file?

No, Desgrana still extracts every channel into a numbered mono file.

The snapshot is used to name the output files after your Wing channels and to detect stereo pairs automatically. If the .snap file is in the same session folder, it is picked up automatically.

My session is split across several WAV files: does that work?

Yes, the Wing creates a new file each time the previous one hits 4 GB (FAT32 limit). Desgrana processes the take files in order and writes one continuous output per channel.

Are session markers preserved?

Markers are embedded as standard cue chunks in every output WAV file, and also exported as a CSV, a plain-text file, and a Standard MIDI File (type 0, SMPTE timecode).

Most DAWs can import the MIDI file to place markers at the correct positions, independent of the project tempo.

Does it send any data online?

No, it works entirely offline.

The only outbound request is an update check once a month, that you may disable in the app settings.

Can I rename tracks before extracting them?

Track names come from your console snapshot (.snap for Wing, .scn for X32/M32). The right place to fix them is either on the console itself, or in your DAW after import.

Desgrana's job is to translate the multitrack WAV into separate, ready-to-use tracks, not to edit the session.
